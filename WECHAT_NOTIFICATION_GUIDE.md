# 企业微信通知功能 - 使用指南

## 📋 概述

本系统集成了企业微信机器人通知功能，可以将螺纹钢期货的市场分析结果、交易信号、重要告警等信息实时推送到企业微信群，方便用户及时获取市场动态。

---

## 🔧 配置说明

### 前置条件

1. **企业微信机器人**
   - 在企业微信群中添加自定义机器人
   - 获取 Webhook URL

2. **环境配置**
   - 系统已集成 `integration-wechat-bot`
   - Webhook key 会自动从系统配置中读取

### 配置步骤

#### 步骤 1: 在企业微信群添加机器人

1. 打开企业微信群
2. 点击群设置
3. 选择"群机器人"
4. 添加机器人
5. 获取 Webhook URL（格式：`https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=XXXXX`）

#### 步骤 2: 在系统中配置

系统会自动读取配置，无需手动设置。配置信息存储在：
- 系统集成配置：`integration-wechat-bot`
- 环境变量：`COZE_WORKLOAD_IDENTITY_API_KEY`

### 2. 发送交易信号

**工具名称**: `send_trading_signal_to_wechat`

**功能**: 发送明确的交易信号通知

**参数**:
- `signal_type`: 信号类型（买入/卖出/观望）
- `price`: 当前价格
- `signal_strength`: 信号强度（强/中/弱）
- `support_level`: 支撑位
- `resistance_level`: 压力位
- `stop_loss`: 止损位
- `take_profit`: 止盈位
- `position_size`: 仓位建议

**使用方式**:
```
请帮我分析螺纹钢期货，如果有明确的买入或卖出信号，请发送到企业微信
```

**返回消息格式**:
```markdown
## 🟢 螺纹钢期货交易信号

**时间**: 2026-03-08 20:30:00

### 信号概要
- **信号类型**: 买入 📈
- **当前价格**: 3067元/吨
- **信号强度**: 中

### 关键价位
- **支撑位**: 3000-3050元/吨
- **压力位**: 3150-3200元/吨
- **止损位**: 3000元/吨
- **止盈位**: 3150元/吨

### 交易建议
- **仓位建议**: 10-20%

---

*⚠️ 以上信号仅供参考，请结合自身判断进行决策*
*🎯 严格执行止损纪律，控制风险*
```

### 3. 发送紧急告警

**工具名称**: `send_alert_to_wechat`

**功能**: 发送系统错误或重要市场事件的告警

**参数**:
- `alert_message`: 告警消息
- `alert_level`: 告警级别（info/warning/error）

**使用方式**:
```
系统出现错误，请发送告警到企业微信
```

**返回消息格式**:
```markdown
## 🚨 交易系统告警

**时间**: 2026-03-08 20:30:00
**级别**: ERROR

**告警信息**:
获取行情数据失败，网络连接超时

请及时查看并处理！
```

---

### 4. 发送每日报告

**工具名称**: `send_daily_report_to_wechat`

**功能**: 发送每日行情汇总报告

**参数**:
- `date`: 日期
- `open_price`: 开盘价
- `high_price`: 最高价
- `low_price`: 最低价
- `close_price`: 收盘价
- `change_percent`: 涨跌幅
- `analysis`: 分析内容

**使用方式**:
```
请帮我生成今天的行情报告并发送到企业微信
```

---

### 5. 发送自定义消息

**工具名称**: `send_to_wechat`

**功能**: 发送自定义文本或 Markdown 消息

**参数**:
- `message`: 消息内容
- `message_type`: 消息类型（text/markdown）

**使用方式**:
```
请把这段文字发送到企业微信：[具体内容]
```

## 🎨 消息格式说明

### Markdown 格式支持

系统使用 Markdown 格式发送消息，支持以下元素：

1. **标题**
   ```markdown
   ## 一级标题
   ### 二级标题
   ```

2. **粗体**
   ```markdown
   **粗体文本**
   ```

3. **列表**
   ```markdown
   - 无序列表项
   1. 有序列表项
   ```

4. **表格**
   ```markdown
   | 列1 | 列2 |
   |-----|-----|
   | 值1 | 值2 |
   ```

5. **图标/表情**
   ```markdown
   📊 📈 📉 ⚠️ 🚨 ✅ ❌
   ```

### 消息长度限制

- Markdown 消息：最长 4096 字节
- 文本消息：最长 2048 字节

## ⚠️ 注意事项

### 1. 通知频率控制

- 避免频繁推送，建议每天不超过 10 次
- 重要信号可以实时推送
- 普通分析建议定时推送（如每天 1-2 次）

### 2. 内容质量保证

- 确保分析结果准确可靠
- 避免发送错误的交易信号
- 重要信息需要人工确认后再推送

### 3. 隐私和安全

- 不要在消息中包含敏感信息（如账户信息、密码等）
- Webhook URL 不要泄露给无关人员
- 定期检查机器人配置

### 4. 失败处理

- 如果发送失败，系统会记录日志
- 可以查看日志排查问题
- 建议设置备用通知方式（如邮件）

## 🔍 故障排查

### 问题 1: 消息发送失败

**可能原因**:
- Webhook URL 配置错误
- 网络连接问题
- 企业微信群已解散或机器人被移除

**解决方案**:
1. 检查 Webhook URL 是否正确
2. 测试网络连接
3. 重新添加机器人并获取新的 Webhook URL

---

### 问题 2: 消息格式显示异常

**可能原因**:
- Markdown 语法错误
- 消息长度超限
- 特殊字符未转义

**解决方案**:
1. 检查 Markdown 语法
2. 缩短消息长度
3. 转义特殊字符

---

### 问题 3: 收不到消息

**可能原因**:
- 企业微信设置了消息免打扰
- 机器人被禁言
- 消息被拦截

**解决方案**:
1. 检查群通知设置
2. 联系群主确认机器人状态
3. 检查消息内容是否包含敏感词

## 📊 最佳实践

### 1. 消息模板化

使用统一的消息模板，确保格式一致：

```markdown
## 📊 [标题]

**时间**: [时间戳]

[内容区]

---

*⚠️ 免责声明*
```

### 2. 优先级分类

根据消息重要性设置不同的通知优先级：

| 优先级 | 消息类型 | 发送频率 | 工具 |
|--------|---------|---------|------|
| 高 | 交易信号 | 实时 | `send_trading_signal_to_wechat` |
| 中 | 每日报告 | 每天 1 次 | `send_daily_report_to_wechat` |
| 低 | 一般分析 | 每天 2-3 次 | `send_market_analysis_to_wechat` |
| 紧急 | 系统告警 | 异常时 | `send_alert_to_wechat` |

### 3. 时间控制

避免在非工作时间发送消息：

- 市场交易时间：9:00-15:00（工作日）
- 收盘总结：15:30-16:00
- 避免在深夜或周末发送非紧急消息

### 4. 内容精简

保持消息简洁明了：
- 使用图标和表格提高可读性
- 关键信息突出显示
- 避免冗长描述

## 🚀 集成示例

### 示例 1: 完整分析流程

```python
# 1. 获取行情
market_data = get_futures_realtime_quotes("螺纹钢")

# 2. 技术分析
analysis = analyze_market(market_data)

# 3. 生成交易建议
recommendation = generate_recommendation(analysis)

# 4. 发送到企业微信
send_market_analysis_to_wechat(analysis)

# 5. 如果有重要信号，发送通知
if recommendation.has_important_signal():
    send_trading_signal_to_wechat(
        signal_type=recommendation.signal,
        price=market_data.current_price,
        ...
    )
```

### 示例 2: 定时推送

```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler()

async def daily_analysis():
    # 分析市场
    analysis = analyze_market()
    
    # 发送报告
    send_daily_report_to_wechat(
        date=datetime.now().strftime('%Y-%m-%d'),
        open_price=analysis.open,
        high_price=analysis.high,
        low_price=analysis.low,
        close_price=analysis.close,
        change_percent=analysis.change,
        analysis=analysis.content
    )

# 每天 16:00 执行
scheduler.add_job(daily_analysis, 'cron', hour=16)
scheduler.start()
```
